# Linear 对象工作台 UX 设计

## 背景

Ameya 当前已经具备项目、资料库、搜索、图谱、时间线、备份、索引、审计、角色成长、模拟、Agent、任务、Prompt 模板、诊断和设置等功能骨架。现有界面更像功能页面集合：左侧主导航承载过多入口，右侧上下文栏仍是占位区，核心编辑表单集中在主内容区，用户在整理资料时容易在功能之间来回跳转。

本设计将 Ameya 调整为面向创作者的专业资料库工作台。一级导航不再按功能分类，而按创作对象分类；右侧栏统一为 Inspector；长内容和复杂编辑进入二级编辑界面。整体视觉参考 Linear 的克制、紧凑、层级清晰和弱装饰风格，但不照搬项目管理工具的信息架构。

## 设计目标

- 将核心体验聚焦到资料库编辑：词条、角色、事件、公理和关系的录入、编辑、关联、审阅。
- 让一级导航表达创作者心智，而不是工程功能清单。
- 充分利用右侧栏，使其成为稳定、可预测、可复用的对象 Inspector。
- 把复杂功能收束到相关对象的二级编辑界面，减少主导航噪音。
- 保持本地优先、Windows 桌面软件、Tauri + Vue 技术栈不变。
- AI、审计、模拟、索引和任务作为辅助能力接入对象上下文，而不是抢占主工作流。

## 设计原则

### 对象优先

用户首先关心的是“我正在处理什么创作对象”，其次才是“我要用什么功能”。因此顶层结构使用创作对象：项目、资料、角色、事件、规则、关系、分析。

### 右栏轻编辑

右侧 Inspector 用于快速判断、轻量编辑和发起动作，不承载长正文和复杂表单。它应该始终回答：当前选中的对象是什么、状态如何、与谁相关、有什么问题、下一步能做什么。

### 二级深编辑

正文、角色档案、事件叙事、规则解释、Prompt 模板、审计报告、模拟报告等复杂内容进入二级编辑界面。二级界面占用主工作区，但保留右侧 Inspector 作为上下文辅助。

### 功能跟随对象

搜索、图谱、时间线、审计、模拟、Agent、索引、任务等能力不再全部作为等权主导航入口。它们根据当前对象出现为视图模式、右栏动作、二级报告或分析工作区。

### 专业软件密度

界面应偏工具型、紧凑、可扫描。避免营销式大标题、过多卡片、单一暖色背景和装饰性图形。优先使用列表、表格、分栏、状态标记、分组标题、键盘操作和命令面板。

### 编辑区优先

布局密度参考 Obsidian 一类专业桌面软件：导航和上下文栏只保留识别、筛选和状态所需的最小尺寸，把横向和纵向空间优先让给中央编辑区。顶部栏、集合栏、Inspector、按钮、表单和卡片都应采用紧凑尺寸；说明性文案不进入主界面，必要信息通过标签、状态、工具提示或二级详情承载。

## 全局信息架构

一级标签位于顶部栏，按创作对象组织：

- 项目
- 资料
- 角色
- 事件
- 规则
- 关系
- 分析

应用级入口不进入一级标签：

- 设置：放在右上角应用菜单和命令面板。
- 帮助：放在应用菜单、命令面板和空状态链接。
- 诊断：放在应用菜单和设置二级页。
- 任务状态：放在顶部右侧和底部状态栏，详情进入分析或任务二级界面。

## 全局布局

### 顶部对象标签栏

顶部栏承载一级对象标签，并尽量利用窗口标题栏附近的横向空间。MVP 阶段先使用应用内顶部栏；后续可评估 Tauri 自定义标题栏、无边框窗口和可拖拽区域。

顶部栏组成：

- 左侧：品牌标识、当前项目名、项目切换入口。
- 中部：对象标签页，展示项目、资料、角色、事件、规则、关系、分析。
- 右侧：全局搜索、命令面板入口、任务状态、应用菜单。

交互规则：

- 点击标签切换当前对象工作区。
- 当前标签下保持最近的集合筛选和视图模式。
- `Ctrl+K` 可跳转到任一对象、记录、命令或设置页。
- 标签不承载过多徽标，只在必要时显示未处理问题数量或任务运行状态。

### 左侧集合栏

左侧栏从主导航降级为当前对象内的集合与筛选。它不再列出所有功能，而是根据顶部标签变化。

示例：

- 资料：全部资料、词条、地点、物品、阵营、资源、草稿、最近编辑、缺少关系。
- 角色：全部角色、主角、配角、按阵营、缺少动机、有关联事件、最近编辑。
- 事件：全部事件、未定时间、按章节、按地点、高影响事件、缺少参与者。
- 规则：全部规则、物理法则、社会规则、魔法规则、例外、低置信度、冲突候选。
- 关系：全部关系、人物关系、因果关系、来源关系、低置信度、孤立对象。
- 分析：搜索、审计、模拟、Agent、索引、任务、报告。

左侧栏宽度应比当前主导航更窄，强调扫描和筛选。集合项应支持数量徽标、选中状态和折叠分组。

### 中央工作区

中央工作区是主要浏览和操作区域。默认形态为列表或表格，允许按对象切换到适合的视图。

基础能力：

- 搜索当前对象集合。
- 排序、筛选、分组。
- 单选、多选和批量操作。
- 新建当前对象。
- 打开二级编辑。
- 根据对象切换列表、表格、时间线、图谱或报告视图。

中央区不应把所有字段全部摊开。它展示用于识别和比较的核心字段，把细节留给右侧 Inspector 和二级编辑。

### 右侧 Inspector

右侧栏统一命名为 Inspector，是所有对象共享的上下文面板。

稳定结构：

1. 标题区：对象名称、类型、状态、更多动作。
2. 快速字段：标签、状态、阵营、时间、重要度、置信度、适用范围等短字段。
3. 摘要区：一句话摘要、最近修改、来源说明。
4. 关系区：反链、关联事件、来源规则、相关角色、目标对象。
5. 质量区：缺失字段、审计提醒、冲突摘要、索引状态。
6. 任务区：相关 AI 任务、审计任务、索引任务和最近操作。
7. 动作区：打开详情、创建关系、运行审计、加入模拟、复制上下文。

不同状态：

- 未选中：展示当前集合的摘要、快捷新建、最近编辑和建议操作。
- 单选：展示该对象 Inspector。
- 多选：展示批量 Inspector，可批量改标签、状态、集合、运行审计或创建任务。
- 无项目：展示项目选择提示和新建项目入口。
- 加载或错误：展示局部状态，不阻塞整个应用。

## 一级标签设计

### 项目

职责：

- 管理项目列表、当前项目、项目元信息。
- 提供导入、导出、备份和项目级统计。
- 作为进入资料库的起点。

左侧集合：

- 最近项目
- 全部项目
- 已归档
- 导入导出
- 备份

中央区：

- 项目列表或密集表格。
- 每行显示项目名、描述、最近编辑、资料数量、问题数量、备份状态。
- 新建项目在顶部工具栏或空状态中完成。

Inspector：

- 项目描述、统计、最近活动。
- 导入导出、备份、归档、打开项目。
- 项目级审计和索引状态。

二级编辑：

- 项目设置。
- 备份详情。
- 导入导出日志。

### 资料

职责：

- 承载词条、地点、物品、阵营、资源等泛世界观条目。
- 是资料库编辑主工作区之一。

左侧集合：

- 全部资料
- 词条
- 地点
- 物品
- 阵营
- 资源
- 草稿
- 最近编辑
- 缺少关系

中央区：

- 默认列表或表格。
- 核心列：标题、类型、状态、标签、关联数量、最近编辑。
- 支持切换到按类型分组。

Inspector：

- 标题、类型、状态、标签。
- 摘要、关联角色、关联事件、来源规则。
- 缺失摘要、孤立对象、潜在冲突提示。
- 动作：打开详情、创建关系、运行审计、生成补完问题。

二级编辑：

- 词条正文。
- 结构化模板字段。
- 版本或编辑历史。
- AI 补完问题结果。

### 角色

职责：

- 管理角色档案、别名、阵营、动机、成长和事件经历。

左侧集合：

- 全部角色
- 主角
- 配角
- 按阵营
- 缺少动机
- 缺少目标
- 有成长记录
- 最近编辑

中央区：

- 角色列表或表格。
- 核心列：姓名、阵营、状态、目标摘要、关联事件、成长状态。

Inspector：

- 姓名、别名、阵营、标签。
- 动机、目标、恐惧的短摘要。
- 关联事件、关系对象、成长趋势摘要。
- 审计提醒：行为矛盾、动机缺失、事件断层。
- 动作：打开角色档案、查看成长、运行行为审计、加入模拟。

二级编辑：

- 完整角色档案。
- 成长分析详情。
- 角色事件经历。
- 行为审计报告。

### 事件

职责：

- 管理时间线、事件详情、参与者、地点、结果和因果链。

左侧集合：

- 全部事件
- 未定时间
- 按章节
- 按地点
- 高影响事件
- 缺少参与者
- 最近编辑

中央区：

- 默认列表；可切换为时间线视图。
- 核心列：标题、时间、地点、重要度、参与者、结果状态。

Inspector：

- 时间、地点、重要度、状态。
- 参与角色、相关资料、触发规则、后续事件。
- 缺失参与者、缺失结果、时间冲突提示。
- 动作：打开事件详情、创建因果关系、运行影响分析、加入模拟。

二级编辑：

- 事件完整描述。
- 起止时间和排序。
- 参与者影响。
- 结果、因果和后续事件。

### 规则

职责：

- 管理公理、世界法则、逻辑约束、例外和适用范围。

左侧集合：

- 全部规则
- 物理法则
- 社会规则
- 魔法规则
- 例外
- 低置信度
- 冲突候选

中央区：

- 规则表格。
- 核心列：主体、谓词、对象、确定性、范围、来源、冲突状态。

Inspector：

- 结构化三元组。
- 适用时间、地点、确定性、来源。
- 被哪些资料、角色、事件引用。
- 冲突摘要和修复建议入口。
- 动作：打开规则详情、创建例外、运行逻辑审计、查看影响范围。

二级编辑：

- 自然语言解释。
- 适用范围和例外。
- 冲突报告。
- 修复方案比较。

### 关系

职责：

- 管理实体间连接、反链、图谱、关系类型和置信度。

左侧集合：

- 全部关系
- 人物关系
- 因果关系
- 来源关系
- 约束关系
- 低置信度
- 孤立对象

中央区：

- 默认关系表格；可切换为图谱视图。
- 核心列：来源、关系类型、目标、方向、置信度、描述。

Inspector：

- 来源对象、目标对象、关系类型、方向、置信度。
- 双方摘要和反链。
- 图谱局部邻域。
- 低置信度或孤立对象提示。
- 动作：打开关系详情、跳转来源、跳转目标、运行关系审计。

二级编辑：

- 关系详情。
- 局部图谱。
- 关系证据和来源。

### 分析

职责：

- 收束搜索、审计、模拟、Agent、索引、任务和报告。
- 分析不是创作对象本身，而是跨对象的辅助工作台。

左侧集合：

- 搜索
- 审计
- 模拟
- Agent
- 索引
- 任务
- 报告

中央区：

- 根据集合切换具体工具界面。
- 搜索结果应可选中并在 Inspector 中显示对象上下文。
- 审计、模拟和任务应以报告列表和状态表格呈现。

Inspector：

- 当前搜索结果、报告、任务或 Agent 上下文。
- 相关对象、输入摘要、输出摘要、状态和后续动作。

二级编辑：

- 审计报告详情。
- 模拟报告详情。
- Agent 对话详情。
- 索引任务详情。
- Prompt 模板管理。

## 二级编辑模型

二级编辑不是新的主导航层，而是当前对象工作区的深编辑状态。

进入方式：

- 双击记录。
- 按 Enter。
- Inspector 中点击打开详情。
- 命令面板跳转。

退出方式：

- 返回列表。
- `Esc` 关闭临时详情。
- 保存后保持在详情，便于连续编辑。

推荐形态：

- 主工作区切换为详情编辑页。
- 左侧集合栏保持当前对象上下文。
- 右侧 Inspector 保持可见，用于关系、审计、任务和快捷动作。

适用内容：

- 资料正文和模板字段。
- 角色完整档案。
- 事件完整叙事和参与者影响。
- 规则自然语言解释和例外。
- 关系证据、描述和局部图谱。
- Prompt 模板。
- 审计报告、模拟报告和任务详情。

## 关键交互规则

### 选择

- 单击记录：选中记录，更新 Inspector。
- 双击记录：进入二级编辑。
- 多选记录：Inspector 切换为批量操作。
- 切换顶部标签：保留该标签上一次集合、筛选和视图状态。

### 新建

- 顶部或中央区工具栏的新增按钮创建当前对象类型。
- 快捷键 `N` 创建当前集合对应对象。
- 在关系对象中新增时，可先选择来源，再选择目标。
- 新建后默认选中记录，并在 Inspector 中填写轻字段；需要长内容时进入二级编辑。

### 保存

- 轻字段可以采用即时保存或小延迟保存，但必须有保存状态反馈。
- 长正文和复杂编辑使用显式保存按钮。
- 保存失败只影响当前对象，不阻塞整个应用。

### 命令面板

命令面板继续作为跨对象入口：

- 跳转对象标签。
- 搜索记录。
- 新建资料、角色、事件、规则、关系。
- 运行审计、模拟、索引。
- 打开设置、帮助、诊断。

### 任务与分析

- 任务状态在顶部右侧和底部状态栏展示。
- 任务详情进入分析工作区。
- 与当前对象相关的任务同时显示在 Inspector。
- AI 功能在无 Provider、无 CLI 或失败时必须降级为可读状态和明确操作建议。

## Linear 风格视觉方向

### 色彩

使用中性底色、细边框、低饱和强调色和明确状态色。避免当前界面偏暖米色和绿色主导的单一色相。建议：

- 应用背景：近白或浅灰。
- 面板背景：白色或轻微灰阶。
- 边框：低对比灰。
- 主强调：冷静蓝或中性黑灰。
- 状态色：成功、警告、错误、运行中分别使用克制色彩，不扩大成大面积背景。

### 密度

- 顶部栏高度紧凑，优先使用标题栏附近横向空间，不制造第二层主导航。
- 左侧集合栏窄而可扫描，集合项以单行标签为主，避免副标题占位。
- 中央列表行高稳定，适合大量资料。
- 右侧 Inspector 分组清晰但克制，默认宽度应小于中央编辑区，避免大卡片堆叠。
- 页面标题不使用 hero 级字号。
- 卡片、按钮、输入框、表单间距采用高密度尺寸，除正文编辑器外不使用大面积留白。
- 窄窗口下，双列或三列的编辑布局应自动收缩为单列，优先保证主编辑区可读与可操作。

### 组件

- 标签页用于一级对象切换。
- 分段控件用于视图模式切换。
- 图标按钮用于搜索、命令、更多、筛选、排序、保存。
- 表格和列表用于对象浏览。
- Inspector 分组使用轻标题、分隔线和小控件。
- 状态使用 pill 或小徽标，但避免到处都是彩色胶囊。

### 动效

- 切换标签和 Inspector 内容使用轻微过渡即可。
- 不使用装饰性大动画。
- 任务运行和索引状态可使用小型进度指示。

| 成长 | 角色标签的二级分析 |
| 模拟 | 分析 > 模拟，角色和事件 Inspector 中提供加入模拟 |
| Agent | 分析 > Agent，当前对象可作为上下文发送 |
| 任务 | 分析 > 任务，顶部和状态栏提供摘要 |
| Prompt 模板 | 分析或设置中的二级管理界面 |
| 帮助 | 应用菜单、命令面板、空状态链接 |
| 诊断 | 应用菜单或设置二级页 |
| 设置 | 右上角应用菜单和命令面板 |

## 分阶段落地

### UX-M1：外壳重构

- 新增顶部对象标签栏。
- 左侧主导航改为当前标签的集合栏。
- 右侧上下文栏改名并重构为 Inspector 骨架。
- 设置、帮助、诊断迁移到应用菜单和命令面板入口。
- 保持现有路由能力，先通过路由映射支撑新结构。

### UX-M2：资料库对象拆分

- 将现有工作台中词条、角色、事件、公理、关系拆成对应对象标签。
- 中央区使用列表或表格展示对象。
- 右侧 Inspector 支持单选、未选中和多选状态。
- 创建和轻字段编辑进入 Inspector。

### UX-M3：二级编辑

- 为资料、角色、事件、规则、关系建立统一详情编辑模型。
- 长正文、完整档案、事件叙事、规则解释、关系证据进入二级编辑。
- 详情页保留右侧 Inspector。

### UX-M4：分析能力收束

- 搜索、审计、模拟、Agent、索引、任务统一进入分析标签。
- 当前对象的分析动作进入 Inspector。
- 审计报告、模拟报告和任务详情形成二级界面。

### UX-M5：视觉 QA 与桌面增强

- 完成 Linear 风格主题。
- 检查桌面和较窄窗口下的布局稳定性。
- 评估 Tauri 自定义标题栏和窗口拖拽区域。
- 完成 Playwright 截图和关键交互 QA。

## 验收标准

- 一级标签始终是创作对象，而不是功能清单。
- 左侧栏只展示当前对象下的集合、筛选和分组。
- 右侧 Inspector 在未选中、单选、多选、错误和加载状态下都有明确内容。
- 长内容和复杂表单不会塞进右栏。
- 设置、帮助、诊断不再占据一级创作对象导航。
- 搜索、图谱、时间线、审计、模拟、Agent、索引和任务能在新结构下找到明确归属。
- 视觉上具备专业软件的紧凑密度、清晰层级和弱装饰风格。
- 无 AI Provider 或 CLI 时，资料库编辑仍完整可用。

## 非目标

- 本设计不改变后端数据模型。
- 本设计不要求立即实现自定义系统标题栏。
- 本设计不引入新的前端框架或组件库。
- 本设计不要求 AI 自动修改用户资料。
- 本设计不覆盖安装包签名、升级策略和大项目性能基准。

## 后续计划

确认本 UX 设计后，再编写具体实施计划。实施计划应明确文件边界、组件拆分、路由映射、测试策略和分阶段验证命令。建议优先规划 UX-M1 和 UX-M2，先完成外壳与资料库对象工作台，再推进二级编辑和分析收束。
